    备份还原（回档）

    还原一个备份，也回档操作

    回档步骤¶

    MCDR 环境¶

    Prime Backup 目前仅支持在 MCDR 环境中进行备份回档

    在 MCDR 环境中回档一个备份，只需执行如下指令：

    !!pb back

    此命令将回档至最新的非临时备份。如需回档至特定备份，可指定备份 ID：

    !!pb back 78

    回档命令支持多种备份 ID 格式：

    格式 示例 说明
    正整数 !!pb back 12 回档至指定 ID 的备份
    ~ 或 latest !!pb back ~ 回档至最新的非临时备份
    相对偏移 !!pb back ~1 回档至最新备份的前一个备份
    相对偏移 !!pb back ~3 回档至最新备份的前三个备份

    回档命令支持以下可选参数：

    参数 说明
    --confirm 跳过确认步骤，直接开始回档
    --fail-soft 在导出过程中跳过导出失败的文件
    --no-verify 不校验导出文件的内容

    示例：

    !!pb back 12 --confirm --fail-soft

    命令行环境¶

    Prime Backup 暂不支持在命令行环境里恢复备份（下次一定！）

    相关配置¶

    回档相关的配置基本位于下述两个小节：

    • 服务器配置，包含在回档时与 MC 服务器的交互命令
    • 备份配置，包含回档时的文件处理规则

    回档流程详解¶

    下面将列出 PB 回档备份过程中的操作流程

    1. 确认阶段
    2. 显示要回档的备份信息
    3. 等待用户确认（除非使用 --confirm 参数）
    4. 用户可通过 !!pb confirm 确认或 !!pb abort 终止
    5. 服务器关闭
    6. 如果服务器正在运行，执行 10s 倒计时（默认配置 command.restore_countdown_sec: 10）
    7. 倒计时期间可以取消回档操作
    8. 停止服务器并等待完全关闭
    9. 回档前的备份
    10. 如果配置了 backup_on_restore（默认值 true），将在此时创建一个备份，以便不时之需
    11. 备份注释为"回档至#X前的自动备份"
    12. 此备份将被标记为临时备份，将在备份清理时特殊处理
    13. 实际回档操作
    14. 回收站机制：将备份目标目录中现有的所有文件移动到临时回收站，确保回档失败时可完全回滚
    15. 保留文件处理：如果配置了 retain_patterns，使用 gitignore 风格模式匹配并隔离要保留的文件
    16. 文件导出：使用多线程并行将备份文件导出到目标目录
    17. 属性恢复：恢复文件权限、时间戳、所有者和符号链接目标
    18. 保留文件恢复：最后将 retain_patterns 保留的文件移回原位置
    19. 服务器重新启动
    20. 如果服务器原本在运行，重启服务器
